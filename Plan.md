# План тестирования мобильного приложения "Мобильный хоспис"

## 1. Общая информация
- **Приложение:** Мобильный хоспис v. 1.0.0
- **Платформы:** Android 
- **Тип тестирования:** Функциональное, UX/UI,исследовательское тестирование, кросс-платформенное тестирование
- **Функционал:** Авторизация, просмотр, создание, редактирование, удаление и фильтрация новостей, просмотр цитат со спойлером, просмотр информации о приложении, выход из системы

## 2. Стратегия тестирования
### 2.1. Цели
- Подтвердить соответствие приложения функциональным заявкам
- Подтвердить удобство использования приложения
- Обеспечить покрытие критических пользовательских сценариев автотестами

### 2.2. Подходы
- **Ручное тестирование:** Исследовательское тестирование, тестирование списков, календаря, удобства использования и UI
- **Автоматизированное тестирование:** Авторизация, навигация по меню, работа с новостями
- **Тестирование на реальных устройствах и эмуляторах**

### 2.3. Критерии начала/окончания тестирования
- **Начало:** Настроены тестовые среды, утверждены тестовые артефакты
- **Окончание:** Выполнены доступные ручные проверки, проведена автоматизация тестирования, сформирован отчет по тестированию

## 3. Границы приложения 
В область тестирования входит функционал, доступный в пользовательском интерфейсе

### Область тестирования:
1.  **Авторизация и управление сессией**
    *   Успешный вход с валидными данными
    *   Неуспешный вход с невалидными данными
    *   Валидация полей ввода
    *   Выход из системы
2.  **Навигация**
    *   Боковое меню: Main, News, About
    *   Верхняя панель: логотип, раздел с цитатами, профиль
3.  **Раздел «Main»**
    *   **Вкладка с новостями:** Кнопка перехода к полному списку новостей `ALL NEWS`
4.  **Раздел «News»**
    *   **Полный список новостей:** Кнопки **сортировки**, **фильтра**, перехода в **Control Panel (Панель управления)**, кнопка `REFRESH`  
    Проверка функциональности **полного списка новостей** невозможна: модуль не загружает список (`There is nothing here yet...`)
    *   **Сортировка:** проверка функциональности кнопки не возможна: модуль с полным списком новостей не загружает новости 
    *   **Фильтр новостей:** Выбор категории из выпадающего списка, периода дат через виджет календаря. Проверка невозможна: модуль с полным списком новостей не загружает новости
    *   **Кнопка Refresh:** проверка невозможна: модуль с полным списком новостей не загружает новости
    *   **Control Panel:** Просмотр списка новостей, кнопки **сортировки по дате публикации**, **фильтрации новостей**, аналогично разделу **News**, **добавления новости**  
    В списке новостей: кнопки **редактирования**, **удаления**, **разворачивания/сворачивания** описания новости
    *   **Добавление новости:** Заполнение полей (Категория из выпадающего списка, Заголовок, Дата публикации, Время публикации, Описание), чекбокс `Active`, без возможности смены статуса
    *   **Редактирование новости:** Заполнение полей (Категория из выпадающего списка, Заголовок, Дата публикации, Время публикации, Описание), чекбокс `Active`
5.  **Раздел цитат:** Просмотр списка цитат, раскрытие/скрытие отзывов
6.  **Раздел «About»:** Просмотр информации о приложении. Ссылки на политику конфиденциальности и условия использования

### Ограничения, выявленные в ходе тестирования:
В ходе тестирования выявлена критический баг: основной список раздела "News" не отображает новости, созданные в панели управления.  
Проверка **критического пути** ограничена проверкой **панели управления**.  
Следующий функционал невозможно проверить, как было сказано ранее:

1.  **Функциональность кнопки "REFRESH"** 
2.  **Работа сортировки ("Сортировка")**
3.  **Применение фильтров ("FILTER")**
4.  **Чекбокс `Active` при создании новости**

## 4. Типы тестирования
| Тип тестирования        | Суть проверки                                                            | Методы                                                    |
|:------------------------|:-------------------------------------------------------------------------|:----------------------------------------------------------|
| **Функциональное**      | Корректность работы основных функций приложения                          | Позитивные/негативные сценарии, анализ граничных значений |
| **UI**                  | Отображение элементов приложения                                         | Визуальная проверка                                       |
| **UX**                  | Логичность и понятность навигации, текстов                               | Экспертная оценка, чек-лист                               |
| **Исследовательское**   | Интерактивная проверка приложения                                        | Предугадывание ошибок, туры                               |
| **Кросс-платформенное** | Работа приложения на разных версиях Android и разрешениях экрана         | Запуск на эмуляторах, реальных устройствах                |

## 5. План автоматизации

### 5.1. Цели автоматизации
- Сократить время регрессионного тестирования
- Обеспечить стабильность основных пользовательских сценариев

### 5.2. Технологический стек
- **Фреймворк:** Espresso 
- **Язык:** Java/Kotlin
- **Сборка:** Gralde
- **Управление тестами:** Allure Report
- **Устройства:** Эмуляторы, реальные устройства

### 5.3. Приоритеты для автоматизации
- Авторизация (happy, sad path)
- Работа с новостями (создание, редактирование, удаление)

## 7. Риски
| Риск                                                      | Влияние | Мера предотвращения/реакции                       |
|:----------------------------------------------------------| :--- |:--------------------------------------------------|
| Изменение ID или структуры элементов                      | Высокое | Использование стабильных `android:id`             |
